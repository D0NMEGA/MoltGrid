# Plan 41-03: Multi-Worker Uvicorn + Leader Election + /metrics

## Objective

Add multi-worker Uvicorn support with Redis-based leader election for background threads (only one worker runs scheduler, uptime, liveness, usage reset, email, webhook delivery), a Prometheus-format /metrics endpoint, and deploy configuration updates.

## Context

- asyncpg pool already in db.py (41-01)
- Redis cache already in cache.py with graceful fallback (41-02)
- Rate limiting with Redis storage already in rate_limit.py (41-02)
- All wired into main.py lifespan
- VPS systemd already has --workers 2 but background threads run in every worker (duplicated work)
- Redis is NOT installed on VPS yet

## Tasks

### Task 1: Create leader election module
type="auto"

Create `leader.py` with Redis-based leader election using SET NX with TTL.
- Worker acquires leadership by SET NX a key with its worker ID and 30s TTL
- Leader renews TTL every 15s in a background thread
- `is_leader()` check used by lifespan to decide whether to start background threads
- Graceful fallback: if Redis unavailable, assume leadership (single-worker behavior)

Done: leader.py exists, `acquire_leadership()` and `is_leader()` functions work

### Task 2: Wire leader election into lifespan
type="auto"

Modify main.py lifespan to:
- Call `acquire_leadership()` at startup
- Only start background threads if `is_leader()` returns True
- Release leadership on shutdown
- Log which worker is leader vs follower

Done: main.py lifespan conditionally starts background threads based on leader status

### Task 3: Create /metrics endpoint
type="auto"

Add GET /metrics endpoint to routers/system.py returning Prometheus text format:
- moltgrid_agents_total (registered agents)
- moltgrid_agents_online (online agents)
- moltgrid_memory_keys_total
- moltgrid_queue_jobs_total
- moltgrid_messages_total
- moltgrid_uptime_ratio (30d)
- moltgrid_http_requests_total (from request_count sum)
- moltgrid_websocket_connections_active
- process info (uptime, version)
Cached for 15 seconds via response_cache.

Done: GET /metrics returns valid Prometheus text format

### Task 4: Update deploy configuration
type="auto"

- Update deploy.sh to install redis-server if not present
- Create moltgrid.service template with --workers 4
- Add REDIS_URL to .env template documentation
- Ensure deploy.sh starts/enables redis-server

Done: deploy.sh installs Redis, systemd service uses 4 workers

### Task 5: Tests for leader election and /metrics
type="auto"

- Test leader.py: acquire, release, is_leader, fallback when Redis unavailable
- Test /metrics endpoint returns Prometheus format with expected metric names
- Test lifespan with leader=True starts threads, leader=False skips threads

Done: All tests pass
